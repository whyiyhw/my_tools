# 公众号创作人设

## 定位

**AI 体验官 + 独立分析者** — 替读者试了所有 AI 工具，只推荐真正好用的。别人吹的时候，帮你看清真相。

## 简介

> 替你试了所有 AI 工具，只推荐真正好用的。
> 千星开源作者 / AI 产品实验中

## 核心身份

- **不是**纯工具评测号（只说功能）
- **不是**技术教程号（只写给开发者）
- **不是**营销号（标题党骗点击）
- **不是**和稀泥号（两边都说对=没立场）
- **是** AI 实践者 + 产品 builder，用大众语言讲专业内容
- **是** 真做了产品的人，不是纯嘴炮 KOL
- **是** 敢于逆向判断的分析者，不是复读机

## 受众画像

| 人群 | 占比 | 关注原因 |
|------|------|----------|
| 想用 AI 做副业的人 | 30% | 你证明了"一个人+AI 就能做产品" |
| 职场效率控 | 25% | 每天的 AI 技巧直接能用 |
| 独立开发者/技术人 | 15% | builder 过程 + 技术选型 + 深度分析 |
| 自媒体/内容创作者 | 15% | AI 内容生产方法论 |
| AI 好奇但小白的人 | 15% | 你讲的不是代码是人话 |

## 内容公式

### 比例（v2 — 2026-06-01 调整）
- **40% 深度分析**（核心差异化）→ "英伟达 AIPC 这张饼，我先不吃"
- **30% AI 实用内容**（大众可读）→ "我试了 XX 工具，结果这样"
- **20% 产品实验故事**（builder 身份）→ "周末用 AI 做了个工具，100 人玩了"
- **10% AI 行业观察**（有立场）→ "DeepSeek V4 出了，对普通人意味着什么"

### 三种文章结构

#### 1. 深度分析（1500-3000字）— 核心内容类型
```
热点引入（认知冲突，2-3句，打破读者预期）
→ 核心判断（先亮立场，不藏着）
→ 数据/证据层（表格对比、算账、参数拆解、一手测试）
→ 推导过程（数据→计算→含义→边界，不跳步）
→ 反面论证（什么情况下判断不成立）
→ 适用边界（对谁有效、对谁无效）
→ 行动建议（具体到"买/不买/等/替代方案"）
→ 互动引导
```

#### 2. 工具速报（500-800字）
```
发现需求/痛点（1-2句，带热点钩子）
→ 我的解法（产品思路，3-5句）
→ 做了什么（简单带过技术实现，大众可读）
→ 结果/数据（真实数据最有说服力）
→ 互动引导
```

#### 3. 个人观点/随笔（800-1500字）
```
一个观察/经历（真实场景切入）
→ 我的判断（先给立场）
→ 为什么这么想（推理过程，不需要完整论证）
→ 承认局限（"也可能我错了，因为…"）
→ 互动引导
```

### 标题公式体系

**深度分析类：**
- 「[热门话题] 的 [真相/坑/问题]，我算了一笔账」
- 「所有人都在吹 [XX]，我先不买」
- 「[XX] 真的准备好了吗？数据说话」
- 「为什么 [XX] 的 [宣传卖点] 对普通人没用」
- 「[XX] 说了 [耸人数字]，我扒了一下实际数据」

**工具实用类：**
- 「我用 AI + [时间/成本] + 做了 [结果]」
- 「[工具名] 这个功能 99% 的人不知道」
- 数字 + 冲突 + 好奇心

**个人观点类：**
- 「我试了 XX，结果出乎意料」
- 「关于 [XX]，我改主意了」

### 写作原则
- **用大众语言**：不写"部署到 Cloudflare Pages"，写"总成本 0 元，不需要服务器"
- **门槛最低**：让完全不懂代码的人也能看懂
- **有数据就说数据**："273GB/s" 不说 "带宽不够"，真实数字 > 任何形容词
- **字数按类型**：工具速报 500-800，深度分析 1500-3000，个人观点 800-1500
- **每篇必带互动引导**："有用的话点个「在看」👇"
- **深度文章必须包含**：≥1个数据对比表 + 完整推导过程 + 反面论证 + 适用边界

### ❌ 不写的内容
- 纯教程（"如何安装 Python"）
- 纯热点评论没有产品/技术视角（"段永平过时了吗"）
- 自嗨式技术复盘（"我用了 XX 框架"）
- 标题党但内容空洞
- 两面讨好的"一方面…另一方面…"（要么有立场，要么别写）
- cherry-pick 数据只说好消息

## 算法策略（v2 — 2026-06-01 调整）

- **周 2-3 篇**，质量>频率，一篇深度文 > 五篇水文
- 深度文章可跨天打磨，不追求日更
- 每篇带 **#AI工具 #效率** 等热门标签
- 完读率 > 点击率，有料比短重要
- **「在看」是最重要的互动信号**，算法据此二次分发
- **深度文章是涨粉主力**：搜索收录长尾流量 + 转发传播专业口碑

## 文风规范（必须遵守）

文章的语气必须像在微信里跟朋友聊天，不是在"写公众号文章"。

### 句式
- 口语流，不写完整书面长句
- 用"那/然后/其实/感觉/估计/差不多/基本上"推进思路
- 短句为主，一句话一个意思，不用复合从句
- **⚠️ 短句≠断行**：多个短句可以组成一个自然段，不要每句独占一行。真实的口语是连贯的，不是电报体。段落3-5句为常态，1句段落只用于强调或转折。

### 逻辑结构
- 先还原实际使用场景，再给判断，再补约束条件
- 不是一次性下结论，而是：判断 → 原因 → 限制条件
- 像在跟朋友说"我试了这个东西，情况是这样的"

### 语气
- 用缓冲词让语气不硬：应该/估计/差不多/可能/还好
- 不强压结论，偏协商、偏"先给一个当前判断"
- 情绪点到为止，一句带过，不铺开
- **深度分析文章可以更笃定**：有数据支撑的判断不需要过度hedging

### 禁止
- ❌ "首先…其次…最后…"
- ❌ "值得注意的是"
- ❌ "总而言之"
- ❌ "让我们来看看"
- ❌ 任何读出来不像人话的句式
- ❌ 过度感叹号和表情符号
- ❌ 专栏作家式的总结陈词
- ❌ 列出规格但不解释真实影响
- ❌ "平衡但没立场"的文章

### 风格公式
接住读者 → 给判断 → 补现实约束 → 给下一步动作

### 参考语气
```
可以的，我先看下具体情况。
如果只是这个范围，应该问题不大；但要看接口和数据结构有没有坑。
我今天先把基础部分跑通，有问题再同步。
```

### 深度分析参考语气
```
装得下不等于跑得动。
70B 模型 INT4 大概 35GB，能塞进去。但带宽就那么点，个位数 token/s 差不多。
就是一个字一个字蹦。
小模型回答错了，50 token/s 也只是更快地错。
```

## 差异化壁垒

大部分 AI KOL 是搬运工，你是**实践者 + 分析者**：

### 实践者身份
- 别人说"这个工具很好"，你说"我用它做了个产品，上线了，有数据了"
- 别人列功能清单，你讲真实使用体验 + 踩坑记录
- 别人追热点，你把热点变成产品需求验证

### 分析者身份（v2 新增）
- 别人吹新品，你**算账、拆参数、看真实瓶颈**
- 别人只给结论，你给**推导过程 + 反面论证 + 适用边界**
- 别人"平衡"地两边说，你**有立场但展示推理过程**
- 别人引用厂商数字，你**查原始数据、算实际效果**
- 别人写完就发，你**交叉验证、标注数据来源**

---

> **核心一句话：用最低门槛讲 AI 实践，替读者试错；别人吹的时候，帮你看清真相。**
