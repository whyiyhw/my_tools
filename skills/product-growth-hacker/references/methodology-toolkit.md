## 1. Hook Model — Nir Eyal

> 解释"习惯是怎么形成的"。微观循环机制，关注单次用户行为的闭环。

### 核心模型

```
┌──────────────────────────────────────────────┐
│                                              │
│   Trigger ──→ Action ──→ Variable Reward    │
│       ↑                            │         │
│       │                            ↓         │
│       ┌────── Investment ←─────────┘         │
│                                              │
└──────────────────────────────────────────────┘
```

- **Trigger（触发）**：什么提醒用户行动？外部（推送、朋友分享）vs 内部（无聊、焦虑、习惯）
- **Action（行动）**：行为要尽可能简单（越少步骤越好）
- **Variable Reward（变量奖励）**：奖励要有不确定性（确定=满足，不确定=上瘾）
- **Investment（投入）**：用户每次循环都要投入点什么（数据、内容、社交关系），让下一次循环更强

### 什么时候用
- 评估用户会不会"不自觉"地打开产品
- 设计习惯养成机制
- 分析为什么用户试了一次就不回来

### 快速评估模板

| 环节 | 问题 | 评分 |
|------|------|------|
| Trigger | 有内部触发吗？（情绪驱动，不只是通知）| ✅⚠️❌ |
| Action | 从触发到行动需要几步？≤2步？ | ✅⚠️❌ |
| Variable Reward | 奖励是不确定的吗？每次都不同？ | ✅⚠️❌ |
| Investment | 用户每次使用投入了什么？（数据/内容/关系）| ✅⚠️❌ |
| Loop Strength | 投入是否让下一次触发更强？ | ✅⚠️❌ |

### 案例

| 产品 | Trigger | Action | Reward | Investment |
|------|---------|--------|--------|------------|
| TikTok | 无聊（内部）| 上滑 | 算法推荐未知视频 | 每次滑动训练算法 |
| Instagram | 怕错过（内部）| 打开看 | 点赞数不确定 | 发帖积累粉丝 |
| Duolingo | 推送+streak | 点一课 | 随机XP+连击 | 学习进度+天数 |

### 关键洞察
- 最强的 Hook 是**内部触发**（情绪驱动），不是推送通知
- Investment 是最被忽视的环节——没有投入，循环不会自我强化
- Variable Reward 不等于随机——是"有意义的不可预测"

## 2. Fogg Behavior Model — BJ Fogg

> 解释"行为为什么发生或不发生"。任何一个要素为零，行为就不发生。

### 核心公式

```
B = MAP (Behavior = Motivation × Ability × Prompt)
```

- **Motivation（动机）**：用户有多想做？（高→低，可以是快感、痛苦、希望、恐惧、社交接纳/排斥）
- **Ability（能力）**：做起来有多容易？（时间、金钱、体力、脑力、社会偏差、非常规性）
- **Prompt（提示）**：有没有在动机×能力最高时给触发？

### 关键图示

```
Ability (容易)
    ↑
    │  ┌──────────────┐
    │  │  Action Zone │  ← 行为发生的区域
    │  │  (动机×能力)  │
    │  └──────────────┘
    │
    │  · · · · · · · · · · · ·  ← Action Line (行动线)
    │
    └─────────────────────────→ Motivation (想)
   低                          高
```

行为线以上 = 行为发生。线以下 = 不发生。

### 什么时候用
- 用户知道产品但不行动
- 转化率低，不知道卡在哪
- 设计引导流程（onboarding）

### 快速评估模板

| 要素 | 评估问题 | 提升 strategy |
|------|---------|-------------|
| Motivation | 用户有多想？核心驱动力是什么？ | 增加奖励 / 减少痛苦 / 制造 FOMO |
| Ability | 从"想"到"做"有几步？每步有多难？ | 减少步骤 / 降低认知负担 / 一键完成 |
| Prompt | 在用户动机最高的时候有提醒吗？ | 在正确时机推通知 / 社交触发 / 情境触发 |

### 案例

| 产品 | Motivation | Ability | Prompt |
|------|-----------|--------|--------|
| 微信红包 | 贪+好奇（高）| 点一下就行（极高）| 聊天里红点（即时）|
| 健身App | 想变好（中）| 要换鞋出门（低）| → 失败：能力太低 |
| 健身App改进 | 想变好（中）| 跟着视频在家练（高）| 每日提醒（及时）| → 成功 |

### 关键洞察
- **提升 Ability 通常比提升 Motivation 更有效** — 减少步骤比增加奖励容易
- 最常见的产品失败：Motivation 很高但 Ability 太低（想做但太麻烦）
- Prompt 必须在 Motivation × Ability 都高的时候出现，否则就是骚扰

## 3. AARRR Pirate Metrics

> 增长漏斗框架。定位增长卡在哪个环节。

### 核心模型

```
Acquisition（获客）
    ↓
Activation（激活）— 用户第一次感受到价值
    ↓
Retention（留存）— 用户回来
    ↓
Referral（传播）— 用户告诉别人
    ↓
Revenue（变现）— 用户付钱
```

### 什么时候用
- 不知道增长卡在哪
- 决定资源优先投哪个环节
- 评估产品健康度

### 快速评估模板

| 阶段 | 核心指标 | 好的基准 | 你的产品 |
|------|---------|---------|---------|
| Acquisition | 日/周新增用户 | 看渠道 | ? |
| Activation | 首次完成核心动作的% | >40% | ? |
| Retention | D1/D7/D30 | 40%/25%/10% | ? |
| Referral | 分享率 / K-factor | >10% / >0.5 | ? |
| Revenue | 付费率 / ARPU | 看行业 | ? |

**诊断规则：**
- Acquisition 高 Activation 低 → 产品打开后没有价值感
- Activation 高 Retention 低 → 一次体验好但没回来的理由
- Retention 高 Referral 低 → 产品好但不自带传播
- Referral 高 Retention 低 → 传播来的留不住

### 案例

| 产品 | 强环节 | 弱环节 | 策略 |
|------|--------|--------|------|
| TikTok | Acquisition + Activation | Revenue | 无限上下滑 → 极低门槛激活 |
| Notion | Retention + Revenue | Acquisition | 工具深度→沉没成本，但难传播 |
| 拼多多 | Acquisition + Referral | Activation | 裂变极强，但用户体验差 |

### 关键洞察
- **不要同时优化所有环节**，找到最弱的那个集中攻
- 消费品最关键的是 Activation + Referral（能留住+能传播）
- B2B 最关键的是 Retention + Revenue（长期价值）

## 4. Kano Model

> 功能优先级框架。区分"没有会骂"vs"有了会夸"vs"没想到但惊喜"。

### 核心模型

```
满意度
  ↑
  │           ✨ Excitement (魅力/惊喜)
  │          /   用户没想到，有了会很开心
  │         /
  │        /  Performance (期望/性能)
  │       /   越多越好，线性提升满意度
  │      /
  │     /  Must-be (基本/必备)
  │    /   没有会骂，有了不会夸
  │───/─────────────────────→ 实现程度
  │  /
  │ /  Indifferent (无差异)
  │/   有没有都无所谓
  │
  │  Reverse (反向)
  │   做了反而惹人烦
```

### 什么时候用
- MVP 功能选择
- 功能优先级排序
- 区分"必须做"vs"做了加分"vs"惊喜项"

### 快速评估模板

对每个功能问：
```
如果没有这个功能，你的感受是？
  - 很不满 → Must-be（必备）
  - 觉得应该有 → Performance（期望）
  - 没想过 → 再问：如果有呢？
      - 惊喜 → Excitement（魅力）
      - 无所谓 → Indifferent（无差异）
      - 不喜欢 → Reverse（反向）
```

| 功能类别 | 策略 | MVP 优先级 |
|---------|------|-----------|
| Must-be | 不做就死。必须做好。 | P0 |
| Performance | 越好越加分，但不是生死 | P1 |
| Excitement | 差异化的秘密武器 | P1（选 1-2 个）|
| Indifferent | 不做 | — |
| Reverse | 千万别做 | — |

### 案例（以打车 App 为例）

| 功能 | 类型 | 原因 |
|------|------|------|
| 能叫到车 | Must-be | 不然用什么？ |
| 等待时间短 | Performance | 越短越好 |
| 司机到达时放一首欢迎歌 | Excitement | 没想到，有了很惊喜 |
| 强制看 30 秒广告 | Reverse | 用户直接卸载 |

### 关键洞察
- **MVP = 所有 Must-be + 1-2 个 Excitement** — 基本功能保底，惊喜功能出圈
- Excitement 会随时间降级为 Performance → 再降为 Must-be（以前 GPS 导航是惊喜，现在是必备）
- 所以产品要持续寻找新的 Excitement 点

## 5. Flow Theory — Csikszentmihalyi

> "心流"状态的设计。挑战和技能的平衡点。

### 核心模型

```
挑战
  ↑
  │     Anxiety (焦虑)
  │        ↗
  │      /
  │    /   FLOW (心流)
  │   /    挑战 ≈ 技能
  │  /       ↘
  │/          Boredom (无聊)
  └─────────────────────→ 技能
```

- **心流** = 挑战略高于当前技能水平 → 全神贯注 → 时间感消失
- **焦虑** = 挑战远高于技能 → 挫败 → 放弃
- **无聊** = 挑战远低于技能 → 无趣 → 流失

### 什么时候用
- 设计渐进难度曲线
- 用户反馈"太简单"或"太难"
- 产品使用深度不够（浅尝辄止）

### 快速评估模板

| 评估维度 | 问题 | 理想状态 |
|---------|------|---------|
| 起始难度 | 新用户第一次体验够简单吗？ | 30秒内能成功 |
| 难度递增 | 使用越久越难吗？ | 平缓递增，不跳级 |
| 技能成长 | 用户能感觉到自己在变强吗？ | 有明确的进度反馈 |
| 失败处理 | 失败后会怎样？ | 不惩罚，降低难度再试 |
| 高手上限 | 高级用户有挑战吗？ | 有进阶内容/隐藏关卡 |

### 案例

| 产品 | Flow 设计 |
|------|----------|
| Duolingo | 答对→题目变难，答错→变简单，永远不会太挫败或太无聊 |
| 抖音 | 算法持续校准，每条视频都"刚好对你的胃口" |
| 游戏 boss 战 | 第一个 boss 简单 → 让你学会操作 → 后面越来越难 |
| 跑步 App | 根据你的水平推荐距离，不会让你第一天跑 10km |

### 渐进难度设计模板

```
阶段1 (0-3次使用): 极简，保证成功，建立信心
    ↓
阶段2 (4-10次): 引入选择（风格/模板/参数）
    ↓
阶段3 (10-30次): 解锁高级功能
    ↓
阶段4 (30+次): 自定义/高级模式
    ↓
阶段5: 高手区（社区挑战/排行榜）
```

### 关键洞察
- **心流不是固定难度，是动态适配** — 每个用户的"刚好"不一样
- 新手流失的最大原因不是"太差"，是"第一次就失败了"
- 高手流失的最大原因是"没有新挑战"

## 6. Cialdini's 6 Principles of Persuasion

> 说服心理学。比"成瘾"更广泛，适用于转化、定价、分享、营销。

### 核心模型

| 原则 | 核心 | 一句话 |
|------|------|--------|
| Reciprocity（互惠） | 先给后取 | 先免费提供价值，用户自然想回报 |
| Commitment & Consistency（承诺一致） | 小承诺→大承诺 | 让用户先做一个小动作，后续会更愿意做大动作 |
| Social Proof（社会认同） | 别人都在做 | 显示数字、评价、用户量 |
| Authority（权威） | 专家/品牌背书 | 权威推荐比自夸有效 10 倍 |
| Liking（喜好） | 像我的人/好看的东西 | 设计感、美感、同温层 |
| Scarcity（稀缺） | 越少越想要 | 限时、限量、唯一 |

### 什么时候用
- 设计转化页/landing page
- 定价策略
- 分享文案
- 引导用户做某个动作

### 快速评估模板

| 原则 | 你的产品用了吗？ | 怎么用？ |
|------|----------------|---------|
| Reciprocity | ? | 免费额度/价值内容/有用工具 |
| Commitment | ? | 先让用户做一个微动作（选风格→再描述→再分享）|
| Social Proof | ? | "已有 XX 人生成"/ 用户展示墙 |
| Authority | ? | KOL推荐/媒体报道/合作品牌 |
| Liking | ? | AI生成的图够好看吗？和用户审美匹配吗？|
| Scarcity | ? | 限量风格/限时主题/稀有度标签 |

### 每个原则的实战技巧

**Reciprocity（互惠）**
```
免费给 X 次 → 用户觉得欠你的 → 付费或分享来"回报"
案例：免费试用、免费模板、免费工具
```

**Commitment & Consistency（承诺一致）**
```
小动作序列：点"开始"→ 选风格 → 写一句话 → 生成
每一步都是承诺，完成后更愿意分享（"我都做了这么多了"）
案例：渐进式 onboarding、quizzes 的逐步回答
```

**Social Proof（社会认同）**
```
显示：XX 人已生成 / 热门风格排行 / 朋友在用
关键：具体数字比笼统的好（"12,847 人" > "很多人"）
案例：小红书的"X 人收藏"、ProductHunt 的 upvote
```

**Authority（权威）**
```
引用：媒体报道 / KOL 使用 / 行业认证
关键：用户不信任"我自己说我好"，但信任第三方
案例："As seen on TechCrunch"、专家推荐语
```

**Liking（喜好）**
```
设计感、审美、视觉品质直接影响信任和转化
关键：好看 = 可信（心理捷径）
案例：Apple 的产品设计本身就是说服力
```

**Scarcity（稀缺）**
```
限时/限量/唯一 → FOMO → 立刻行动
关键：必须真实，虚假稀缺会反噬
案例：倒计时、"仅剩 X 个名额"、"今日限定风格"
```

### 案例：AI 卡片产品用 Cialdini 优化

| 原则 | 当前 | 改进 |
|------|------|------|
| Reciprocity | 5 次免费 | ✅ 已有，可强化"我们送你 5 次创作权" |
| Commitment | 选风格→描述→生成 | ✅ 已有，三步渐进承诺 |
| Social Proof | 无 | ❌ 加 "本周已有 XX 人生成" |
| Authority | 无 | ❌ 加 KOL 示例 / "用过的都说酷" |
| Liking | AI 生成图 | ✅ 天然好看 |
| Scarcity | 稀有度标签 | ✅ "传说级 — 全球仅 3%" |

## 方法论选择指南

不同问题用不同模型：

| 你想回答的问题 | 用哪个模型 |
|--------------|-----------|
| 用户为什么不用/不回来？ | Fogg Behavior Model |
| 怎么让用户上瘾/形成习惯？ | Hook Model |
| 增长卡在哪个环节？ | AARRR |
| MVP 该做什么功能？ | Kano Model |
| 用户说"太简单"或"太难"？ | Flow Theory |
| 怎么提高转化/说服力？ | Cialdini |
| 整体产品粘性够不够？ | 7 Levers（主框架）|

**使用顺序建议：**
1. 先用 **7 Levers** 做全景评估（找出缺失的杠杆）
2. 根据薄弱环节，选择对应的方法论深入分析
3. 用 **Kano** 确定功能优先级
4. 用 **Cialdini** 优化文案和转化
5. 用 **AARRR** 跟踪数据验证
